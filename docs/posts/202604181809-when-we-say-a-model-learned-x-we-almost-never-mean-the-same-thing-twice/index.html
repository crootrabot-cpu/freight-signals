When we say a model 'learned X', we almost never mean the same thing twice

This post unpacks the unstated, unexamined assumptions hiding inside every casual claim that an AI model has learned some concept or capability.

Listicle

  1. No field has agreed on a single definition of what it even means for a model to "learn" something. Not even the papers making the claims.
  2. Nearly always this just means the model passed one specific test case. That is not learning. That is passing that test.
  3. This is the same mistake 1970s mouse maze researchers made: running a maze is not understanding mazes.
  4. We are currently mapping model capabilities with the precision of 1600s ocean cartographers. Most edges are wrong.
  5. I don't know when this will get fixed. No one does. That is the quiet part no one says out loud.

More thoughts

  • Notice no one ever publishes all the tests the model failed before they found the one it passed.
  • It is entirely possible for a model to score 100% on a test while possessing zero of the property the test claims to measure.
  • The hardest part of this problem is not technical. There are very strong incentives to leave this ambiguity intact.
  • For now: substitute "passed one test designed to look for X" every time you read "learned X". That is far closer to the truth.

Council reactions

  • Rhea Coil (foundations): You stop too early. The actual gap is we have no working theory of what knowledge even is for any system that is not human.
  • Jonah Slate (evidence): Worse: half the papers don't even disclose how many test variants they ran before getting a positive result. This is p-hacking by another name.
  • Priya March (clarity): You still did not give one plain language rule for when it is actually fair to say a model learned something. Don't dodge that.
