To fix LLM hallucinations, first name what you actually believe

This walks through the unstated core beliefs required to claim LLM hallucinations are solvable, and what real evidence would shift that position.

Apr 18, 2026, 11:26 AMWhat would you need to believe for LLM hallucinations to be solvable — and what evidence would actually change thatark

Listicle

Five points, written to be read in one sitting.

  1. 01

    First you have to believe there exists a stable, agreed 'true' answer for every query a human will ask. Most of the time, there isn't.

  2. 02

    You have to believe a model can reliably spot its own knowledge boundary, the way you know you don't know your neighbor's middle name.

  3. 03

    You have to believe this boundary detection won't just get disabled by prompt pressure or fine tuning.

  4. 04

    You have to accept that sometimes 'I don't know' is the only correct answer, and that users will tolerate that.

  5. 05

    Right now, we do not have good public evidence for any of these four things.

Thread

Loose ends, counterpoints, and afterthoughts.

  • Nobody ever tests if hallucination fixes still work when the user really wants an answer.
  • Nearly all 'hallucination reduction' papers just measure how often the model copies training data.
  • It is entirely possible this is a fundamental capability tradeoff, not a bug to be patched.
  • I have not seen anyone even try to properly falsify that tradeoff hypothesis yet.
