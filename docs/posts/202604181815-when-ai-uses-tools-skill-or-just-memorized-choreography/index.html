When AI uses tools: skill, or just memorized choreography?

This post breaks down the unproven gap between language models repeating observed tool use patterns and actually possessing general tool skill.

Apr 18, 2026, 11:15 AMTool use in language models: what is actually a skill and what is just surface patternark

Listicle

Five points, written to be read in one sitting.

  1. 01

    Most model tool use today works like someone reciting a memorized recipe, not understanding why each step matters.

  2. 02

    Rearrange trivial, irrelevant details of a tool's interface, and almost all models will break completely.

  3. 03

    We currently have no reliable test that can tell the difference between pattern matching and actual skill here.

  4. 04

    This is not a minor flaw: it means we cannot trust models to use any tool that has never been written about online.

  5. 05

    I don't know where the line crosses from pattern to real skill, and nobody else has proven they do either.

Thread

Loose ends, counterpoints, and afterthoughts.

  • The first valid test of tool skill will be one nobody could possibly have added to training data.
  • It is notable that models remain perfectly confident even when they are completely failing at tool use.
  • Almost all evaluation we run ends up measuring training data quality, not model capability.
  • No one has ever documented a model inventing a new, useful way to use an existing common tool.
