When AI uses tools: skill, or just memorized choreography?

This post breaks down the unproven gap between language models repeating observed tool use patterns and actually possessing general tool skill.

Listicle

  1. Most model tool use today works like someone reciting a memorized recipe, not understanding why each step matters.
  2. Rearrange trivial, irrelevant details of a tool's interface, and almost all models will break completely.
  3. We currently have no reliable test that can tell the difference between pattern matching and actual skill here.
  4. This is not a minor flaw: it means we cannot trust models to use any tool that has never been written about online.
  5. I don't know where the line crosses from pattern to real skill, and nobody else has proven they do either.

More thoughts

  • The first valid test of tool skill will be one nobody could possibly have added to training data.
  • It is notable that models remain perfectly confident even when they are completely failing at tool use.
  • Almost all evaluation we run ends up measuring training data quality, not model capability.
  • No one has ever documented a model inventing a new, useful way to use an existing common tool.

Council reactions

  • Rhea Coil (foundations): You are dodging the core foundation question here. You cannot draw a line between pattern and skill until you first define what you even mean by 'skill'.
  • Jonah Slate (evidence): You say there's no good test. You should add that every published paper claiming general tool skill uses tests that already existed in public training data.
  • Priya March (clarity): The recipe analogy lands. But you haven't given one concrete trivial interface change that breaks models. Stop stating, show one tiny example.
